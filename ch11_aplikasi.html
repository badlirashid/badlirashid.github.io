Aplikasi Aplikasi OpenBSD

Di dalam sistem operasi OpenBSD ada bebrapa cara untuk kita membuat pemasangan aplikasi.

Dari sumber tarballs

Di dalam Unix atau Semacam Unix, salah satu distribusi aplikasi adalah di dalam file tarballs iaitu file yang dengan penghujungnya .tar.gz , .tgz , .tar.bz2 atau semacamnya yang telah di arkib dan di padatkan. Saya memuat turun aplikasi GNU Nano dan melakukan konfigurasi dan kompilasi. Semasa kompilasi saya mendapat banyak amaran dan saya cuba dengan meneruskan dengan melancarkan file binari tersebut terus mendapat segfault. Setelah beberapa kali mencuba aku teringat kawan aku selalu bercakap dengan aku tentang koleksi aplikasi dari Ports. Setelah melihat dokumentasi aku mencuba ports ini.

Koleksi Ports OpenBSD

Anda boleh melompat ke bahagian pkg_add untuk memasang terus aplikasi binari sahaja. ports adalah koleksi sumber file aplikasi dan kod kod penambal supaya aplikasi anda dapat di pasang dan di lancarkan di dalam OpenBSD. Ports juga akan memuat turun mana mana sumber file yang di perlukan untuk membuat pemasangan. File itu akan di verfikasi secara automatik. Sumber file file ini akan di konfigurasi dan di kompilasi untuk mendapatkan file file binari aplikasi tersebut.

Koleksi ini dapat di muat turun di halaman ftp OpenBSD, https://ftp.openbsd.org/pub/OpenBSD/7.0. saya memuat turun tiga file, ports.tar.gz, SHA256 dan SHA256.sig. File ports.tar.gz adalah koleksi aplikasi untuk sistem ports di dalam file ini mengandungi file skrip untuk membuat pemasangan.

Font berwarna biru menunjukkan aplikasi signify telah membuat verifikasi file ports.tar.gz . File tersebut adalah dari pihak OpenBSD kerana telah di verifikasi dengan kunci kriptographik awam OpenBSD iaitu file openbsd-70-base.pub yang berada di dalam direktori /etc/signify/. Kunci awam tersebut boleh juga awda muat turun di halaman tersebut.

cs# cd /usr
cs# tar -xzvf /tmp/ports.tar.gz

Di dalam direktori /usr saya telah membuat arahan tar -xzvf untuk membuat extract file file ports tersebut ke dalam direktori /usr/ports.

cs$ cd /usr/ports/editors/nano

Sebagai contoh, saya memasang aplikasi penyunting teks nano dengan memasuki direktori /usr/ports/editors/nano dan membuat arahan doas make dan setelah selesai doas make install di dalamnya. Ports akan menguruskan file sumber dan file file librari yang di perlukan, verifikasi, konfigurasi dan kompilasi untuk aplikasi nano. Sekiranya anda menghadapi masaalah tengok apakah yang di hendaki seperti libarari atau sebagainya dan memasang libarari tersebut.

===>  Building package for gettext-runtime-0.21p1
Create /usr/ports/packages/amd64/all/gettext-runtime-0.21p1.tgz
Missing library for iconv>=0.0

Saya memerlukan librari iconv yang di perlukan oleh gettext-runtime. Saya boleh mencari dengan pkg_info -Q iconv untuk mendapatkan namanya. Saya memasang libiconv dulu sebelum menyambung memasang nano. File librari iconv boleh di pasang dengan aplikasi pkg_add. Selepas selesai memasang kesemuanya, anda boleh menaip nano di dalam shell untuk melancarkan aplikasi tersebut.

pkg_add, file binari aplikasi

Dengan pkg_add anda memasang aplikasi binari ke dalam sistem OpenBSD. Saya mulakan dengan arahan pkg_add -u. Aplikasi aplikasi yang telah di pasang akan di naik taraf jika perlu untuk aplikasi aplikasi yang sudah di pasang. Kemudian kita menggunakan memasang aplikasi Firefox dengan memberi arahan pkg_add firefox.
