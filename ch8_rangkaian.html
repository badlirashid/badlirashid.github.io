Aplikasi Rangkaian

ifconfig

ifconfig adalah untuk melihat status perkakas kad rangkaian, perkakas modem dan kofigurasi perkakas rangkaian tersebut. Di dalam windows ia di panggil ipconfig.

cs# ifconfig
lo0: flags=8049<UP,LOOPBACK,RUNNING,MULTICAST> mtu 32768
        index 3 priority 0 llprio 3
        groups: lo
        inet6 ::1 prefixlen 128
        inet6 fe80::1%lo0 prefixlen 64 scopeid 0x3
        inet 127.0.0.1 netmask 0xff000000
em0: flags=248843<UP,BROADCAST,RUNNING,SIMPLEX,MULTICAST,AUTOCONF6TEMP,AUTOCONF6> mtu 1500
        lladdr 08:00:27:2e:b0:c4
        index 1 priority 0 llprio 3
        groups: egress
        media: Ethernet autoselect (1000baseT full-duplex)
        status: active
        inet6 fe80::a00:27ff:fe2e:b0c4%em0 prefixlen 64 scopeid 0x1
        inet 192.168.56.101 netmask 0xffffff00 broadcast 192.168.56.255
enc0: flags=0<>
        index 2 priority 0 llprio 3
        groups: enc
        status: active
pflog0: flags=141<UP,RUNNING,PROMISC> mtu 33136
        index 4 priority 0 llprio 3
        groups: pflog

Dari ifconfig boleh menujukan bebrapa butiran tentang perkakas kad rangkaian, font berwarna biru menunjukan butiran seperti:-

  • kad rangkaian yang bernama em0.
  • Kad in berstatus active
  • IPv6 telah di konfigure dengan AUTOCONF6.;
  • Alamat MAC perkakas ini adalah 08:00:27:2e:b0:c4.
  • Rangkaian kita bejalan dengan rangkaian Gigabit, Ethernet autoselect (1000baseT full-duplex)
  • Alamat IPv6 kita adalah fe80::a00:27ff:fe2e:b0c
  • Dan alamat IPv4 kita adalah 192.168.56.101.

route

route adalah aplikasi yang melihatkan di mana packet packet network kita akan di salurkan. Selain itu route juga dapat memanupilasi route table kita seperti menambah, menghapuskan dan mengubah routing table tersebut. Di dalam Openbsd route utama dapat di lihat di dalam file /etc/mygate sekiranya kita menggunakan protokol DHCP kita pakai route show.

route show menunjukan bahawa semua rangkaian packet di salurankan ke saluran utama iaitu Gateway yang beralamat 192.168.56.1. Sekiranya perkataan Default dari pada Destinasi tidak ada di dalam routing table lihat IP yang beralamatkan 0.0.0.0.

ping6 menggunakan alamat IPv6 dan kad rangkaian. Seperti contoh di atas font bewarna biru, ia adalaah % dan kad rangkaian em0 selepas alamat IPv6.

arp dan ndp

arp adalah untuk IPv4 dan ndp adalah untuk IPv6. Dua program yang sama fungsi ia menunjukan alamat IP ke alamat MAC dan boleh mengubah mapping table tersebut

cs$ arp -a
Host                                          Ethernet Address        Netif Expire       Flags
192.168.131.11                        08:00:27:79:34:28        em0   11m57s
192.168.131.50                        08:00:27:2e:b0:c4        em0   permanent   l

Contoh menggunakan arp, resolusi alamat IPv4 ke alamat MAC

cs$ ndp -a
Neighbor                                              Linklayer Address    Netif Expire          S Flags
fe80::a00:27ff:fe2e:b0c4%em0          08:00:27:2e:b0:c4     em0  permanent     R l
fe80::a00:27ff:fe79:3428%em0          08:00:27:79:34:28     em0  23h56m18s   S

Contoh menggunakan program ndp, resolusi alamat IPv6 ke alamat MAC
