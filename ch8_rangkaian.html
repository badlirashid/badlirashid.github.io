Aplikasi Rangkaian

Aplikasi aplikasi rangkaian ini adalah yang terdapat di dalam OpenBSD 7.0. Penjelasan hanya ringkas sahaja, kalau hendak lebih butiran lanjut kita boleh mengunakan aplikasi man untuk melihat dokumentasi tersebut.

ifconfig

ifconfig adalah untuk melihat status perkakas kad rangkaian, perkakas modem dan konfigurasi perkakas rangkaian tersebut. Di dalam windows ia di panggil ipconfig.

cs# ifconfig
lo0: flags=8049<UP,LOOPBACK,RUNNING,MULTICAST> mtu 32768
        index 3 priority 0 llprio 3
        groups: lo
        inet6 ::1 prefixlen 128
        inet6 fe80::1%lo0 prefixlen 64 scopeid 0x3
        inet 127.0.0.1 netmask 0xff000000
em0: flags=248843<UP,BROADCAST,RUNNING,SIMPLEX,MULTICAST,AUTOCONF6TEMP,AUTOCONF6> mtu 1500
        lladdr 08:00:27:2e:b0:c4
        index 1 priority 0 llprio 3
        groups: egress
        media: Ethernet autoselect (1000baseT full-duplex)
        status: active
        inet6 fe80::a00:27ff:fe2e:b0c4%em0 prefixlen 64 scopeid 0x1
        inet 192.168.56.101 netmask 0xffffff00 broadcast 192.168.56.255
enc0: flags=0<>
        index 2 priority 0 llprio 3
        groups: enc
        status: active
pflog0: flags=141<UP,RUNNING,PROMISC> mtu 33136
        index 4 priority 0 llprio 3
        groups: pflog

Dari ifconfig boleh menujukan bebrapa butiran tentang perkakas kad rangkaian, font berwarna biru menunjukan butiran seperti:-

  • kad rangkaian yang bernama em0.
  • Kad in berstatus active
  • IPv6 telah di konfigurasi dengan AUTOCONF6.;
  • Alamat MAC perkakas ini adalah 08:00:27:2e:b0:c4.
  • Rangkaian kita berjalan dengan rangkaian Gigabit, Ethernet autoselect (1000baseT full-duplex)
  • Alamat IPv6 kita adalah fe80::a00:27ff:fe2e:b0c
  • Dan alamat IPv4 kita adalah 192.168.56.101.

route

route adalah aplikasi yang melihatkan di mana packet packet network kita akan di salurkan. Selain itu route juga dapat memanipulasi route table kita seperti menambah, menghapuskan dan mengubah routing table tersebut. Di dalam Openbsd route utama dapat di lihat di dalam file /etc/mygate sekiranya kita menggunakan protokol DHCP kita pakai route show.

route show menunjukan bahawa semua rangkaian packet di salurankan ke saluran utama iaitu Gateway yang beralamat 192.168.56.1. Sekiranya perkataan Default dari pada Destinasi tidak ada di dalam routing table lihat IP yang beralamat kan 0.0.0.0.

ping dan ping6

64 bytes from 192.168.131.11: icmp_seq=2 ttl=255 time=0.832 ms
64 bytes from 192.168.131.11: icmp_seq=3 ttl=255 time=0.382 ms

--- 192.168.131.11 ping statistics ---

ping6 menggunakan alamat IPv6 dan kad rangkaian. Seperti contoh di atas font berwarna biru, ia adalah % dan kad rangkaian em0 selepas alamat IPv6.

arp dan ndp

arp adalah untuk IPv4 dan ndp adalah untuk IPv6. Ke dua dua aplikasi ini sama fungsinya iaitu menunjukkan alamat IP ke alamat MAC dan boleh mengubah mapping table tersebut.

cs$ arp -a
Host                                          Ethernet Address        Netif Expire       Flags
192.168.131.11                        08:00:27:79:34:28        em0   11m57s
192.168.131.50                        08:00:27:2e:b0:c4        em0   permanent   l

Contoh menggunakan arp, resolusi alamat IPv4 ke alamat MAC

cs$ ndp -a
Neighbor                                              Linklayer Address    Netif Expire          S Flags
fe80::a00:27ff:fe2e:b0c4%em0          08:00:27:2e:b0:c4     em0  permanent     R l
fe80::a00:27ff:fe79:3428%em0          08:00:27:79:34:28     em0  23h56m18s   S

Contoh menggunakan program ndp, resolusi alamat IPv6 ke alamat MAC

traceroute dan traceroute6

Fungsi aplikasi traceroute ini adalah untuk melihat di mana laluan packets yang kita hantar berjalan di dalam rangkaian untuk menuju ke destinasinya.

cs$ traceroute 192.168.141.1
traceroute to 192.168.141.1 (192.168.141.1), 64 hops max, 40 byte packets
 1  router (192.168.131.11)  2.011 ms  0.675 ms  0.551 ms
 2  router (192.168.131.11)  0.483 ms !H  0.637 ms !H  0.337 ms !H

Di atas menunjukkan traceroute untuk IPv4.

cs$ traceroute6 fe80::a00:27ff:fe79:3428%em0
traceroute6 to fe80::a00:27ff:fe79:3428%em0 (fe80::a00:27ff:fe79:3428%em0), 64 hops max, 60 byte packets
 1  router (fe80::a00:27ff:fe79:3428%em0)  4.735 ms  0.743 ms  0.708 ms

Di atas menunjukkan traceroute untuk IPv6

host

Aplikasi ini menunjukkan nama penuh domain ke alamat IP sahaja. Contoh host www.openbsd.org.

cs# host www.openbsd.org
www.openbsd.org has address 199.185.178.80

nslookup

Aplikasi ini adalah untuk melihat butiran rekod domain di DNS server. Sekiranya kalau di gunakan secara langsung ia akan menunjukkan nama domain penuh ke alamat IP. Contoh nslookup www.openbsd.org.

cs# nslookup www.openbsd.org
Server:         8.8.8.8
Address:        8.8.8.8#53

Non-authoritative answer:
Name:   www.openbsd.org
Address: 199.185.178.80

netstat

netstat adalah aplikasi untuk melihat status rangkaian kita yang sedang aktif.

/etc/hosts

Bukannya aplikasi atau skrip, file hosts ini mengandungi nama hos ke alamat IP secara manual.
