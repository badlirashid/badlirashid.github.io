File dan Direktori Akses

Sistem Unix Bermula Dengan /

Dalam sistem Unix dan yang semacamnya, permulaan bermula dengan / atau slash root. Di sini semua direktori utama akan bermula dan dapat dilihat. Sistem ini tidak membezakan lokasi dengan berasaskan partitions atau perkakas storage. Seperti pembaca cakera padat ia dapat di akeses dengan /mnt/cdrom direktori.

Berbeza dengan Windows ia bermula dengan C:\ (C Drive) mahupun ia satu fisikal cakera keras atau satu partion. Jikalau ada partion kedua atau pembaca cakera padat ia akaan di namakan sebagai D:\ (D Drive).

Berikutnya menunjukan butiran file dan direktori yang ada di dalam permasang OpenBSD 7.0 saya.

Nama File dan Direktori

Nama nama di dalam OpenBSD adalah case sensitive. Ini bermaksud nama itu berbeza jikalau kita menggunakan huruf besar ataupun kecil. Maksud saya jikalau dua folder bernama "Artikel1" dan "artikel1" ia adalah berbeza kerana huruf a and A adalah perbezaannya. Ini sama seperti dengan file bernama "oren.txt" dan "orEn.txt", file itu berbeza kerana huruf "e".

Akses

Di dalam Unix, untuk membahagi Pengguna (Users) adalah mengikut Kumpulan (Group).

File dan direktori akses boleh di bahagi kepada tiga jenis iaitu:-

  • baca (read)
  • tulis (write)
  • dan lancarkan (execute) program atau skrip

Butiran File

cheesysalmon$ touch filesaya
cheesysalmon$ ls -al filesaya
-rw-r--r-- 1 badli badli 0 Feb 26 13:40 filesaya

Pertama saya membuat file kosong bernama filesaya dengan program touch.

Dengan menggunakan "ls -al", butiran akses "filesaya" ialah "-rw-r--r--". Akses itu di beri kepada tiga kategori iaitu Pemilik (owner), Kumpulan (group) dan yang lain (others). Akses di baca dari kiri ke kanan, dari numbor dua dan di bagi kepada tiga kategori.

Pemilik (owner) di beri akses baca dan tulis (rw-),
kumpulan (group) di beri akses baca sahaja (r--)
dan yang lain (others)di beri akses baca sahaja (r--).

File "filesaya" itu di miliki oleh badli dan group file itu ada lah juga badli. File ini di punyaie oleh saya dan akses kepada yang lain adalah membaca.

Butiran Direktori

Untuk membuat direktori baru pula, kita gunakan arahan "mkdir foldersaya". Apa bila kita beri arahan ls -al butiran directori tersebut seperti berikut:-

cheesysalmon$ mkdir foldersaya
cheesysalmon$ ls ‐al
drwxr-xr-x  2 badli  badli  512 Feb 26 14:03 foldersaya

"d" bermaksud direktori dan ini membezakan jenis dengan file.
Akses pemilik adalah baca, tulis dan lancarkan rwx, untuk kumpulan ada lah baca dan lancarkan r-x, dan yang lain adalah baca dan lancarkan r-x.

"x" adalah penanda untuk membuat program atau skrip dapat di lancarkan. Tetapi untuk direktori pula, ia di gunakan supaya kita dapat memasuki direktori "foldersaya".

Mengubah Butiran

Untuk mengubah pemilik atau kumpulan kita menggunakan program "chown" maksud nya adalah "change owner". Bagi mengubah akses file pula kita akan mengunakan program "chmod" iaitu "change mode".

Untuk chmod kita menggunakan angka untuk menukar akses tersebut. 4 untuk akses baca, 2 untuk akses tulis and 1 untuk akses melancar. Angka angka tersebut akan di campur untuk membagi akses file tersebut. Sebagai contoh untuk website kita akan tukar akeses file html kepada akses baca kepada kumpulan dan lain lain.

cheesysalmon# ls -al index.html
-rwxrwxrwx 1 badli badli 0 Feb 26 19:59 index.html
cheesysalmon# chmod 644 index.html
cheesysalmon# chown badli.www index.html
cheesysalmon# ls -al index.html
-rw-r--r-- 1 badli www 0 Feb 26 19:59 index.html
cheesysalmon#

Contoh di atas akan menukar akses dan kumpulan untuk file index.html tersebut. Prompt yang dulunya $ betukar ke # kerana program program tersebut di jalankan dengan pengguna root.

cheesysalmon# ls -al
drwxr-xr-x 2 badli badli 512 Feb 26 19:59 direktori_website
cheesysalmon# chown root.wheel direktori_website
cheesysalmon# chmod 700 direktori_website
cheesysalmon# ls -al
cheesysalmon$ exit
drwx------ 2 root wheel 512 Feb 26 19:59 direktori_website
cheesysalmon$ cd direktori_website/
ksh: cd: /var/www/direktori_website - Permission denied
cheesysalmon$ su -
Password:
cheesysalmon# cd /var/www/htdocs
cheesysalmon# chown badli.www direktori_website
cheesysalmon# chmod 755 direktori_website
cheesysalmon# ls -al
drwxr-xr-x 2 badli www 512 Feb 26 19:59 direktori_website
cheesysalmon# exit
cheesysalmon$ cheesysalmon$ cd /var/www/htdocs/direktori_website
cheesysalmon$ mkdir test_dir
cheesysalmon$

Contoh di atas saya, mendapat telah "Permission denied" ketika memasuki direktori_website. Apabila saya merubah akses tersebut saya dapat memasuki direktori_website dan boleh membuat direktori baru didalamnya bernama test_dir
