Pemasangan OpenBSD

Sebelum anda memasang OpenBSD ke dalam Laptop, Desktop atau Server. Sebaiknya kita selidiki jikalau perkakas anda serasi dengan OpenBSD. Maksud perkakas di sini ialah:-

  • Storage controller
  • Network Adapter
  • Wireless Adapter
  • Graphics Card
  • Audio Card
  • dan perkakas perkakas yang lain.

Berbanding perkembangan Linux drivers, OpenBSD berada setapak di belakang. Penggunaan utama sistem operasi OpenBSD adalah di gunakan untuk Server, Router dan Firewall tetapi ada juga yang memasang untuk penggunaan harian ke dalam Laptop dan Desktop mereka.

Virtualisasi

Sekiranya anda belum selesa dengan pemasang OpenBSD ke dalam Laptop, Desktop atau Server anda boleh memasang sistem operasi tersebut secara virtual. Contoh aplikasi virtualization yang ada adalah seperti berikut:-

  • vmm
  • bhyve
  • Linux KVM
  • Xen Project
  • Oracle VirtualBox
  • Microsoft Hyper-V
  • dan VMware

Media Permasangan

Installation media boleh didapati di halaman utama OpenBSD. Media yang boleh digunakan seperti USB Pendrive, Cakera Padat dan Floppy.

Ada berapa versi media pemasangan yang tersedia mengikut jenis Processor. Saya cuma memilih tiga sahaja:-

  • i386 - Processor 32bit untuk Intel i386 atau yang sebanding dengannya.
  • amd64- Processor 64 bit Dari AMD64 atau yang sebanding dengannya.
  • macppc - Processor Power PC telah digunakan oleh Apple sebelum mereka bertukar ke Intel Processor dahulu

Processor 64bit dan 32bit

Sekiranya anda memasang drivers perkakas untuk sistem 32 bit, drivers itu tidak akan berfungsi ke sistem 64 bit dan begitu juga sebaliknya. Aplikasi OpenBSD 32 bit tidak boleh dipakai host OpenBSD 64 bit dan aplikasi OpenBSD 64bit tidak boleh dipakai host OpenBSD 32bit.

Maklumat sebelum pemasangan

Sebelum memasang perisai ada beberapa maklumat anda perlu ada terlebih dahulu seperti:-

  • hostname ia itu nama komputer anda
  • IP address
  • IP address DNS

Dokumentasi untuk memasang file-file pemasangan OpenBSD ke dalam media yang anda pilih ada di dalam halaman utama mereka. Masukkan media tersebut and mulakan pengembaraan anda ke dalam alam OpenBSD.

1 - Boot Dari Pemasangan Media

>OpenBSD\amd64 CDBOOT 3.53
boot>

boot Cakera Padat yang mengandugi file permasangan OpenBSD

Sistem menanti arahan untuk membuat boot ke cakera padat OpenBSD. Tekan hanya menekan "Enter" untuk meneruskan.

2 - Skrip pemasangan OpenBSD

scsibus1 at softraid0: 256 targets
root on rd0a swap on rd0b dump on rd0b
WARNING: CHECK AND RESET THE DATE!
erase ^?, werase ^W, kill ^U, intr ^C, status ^T

Welcome to the OpenBSD\amd64 7.0 installation program.
(I)nstall, (U)pgrade, (A)utoinstall or (S)hell?

Semasa process Boot, ia akan menunjukkan perkakas yang ada terpasang di dalam sistem anda. Setelah selesai ia menunjukkan beberapa pilihan. Tekan huruf "i" and tekan "Enter" untuk meneruskan.

root dan Pengguna biasa

OpenBSD mempunyai dua jenis pengguna, root akaun dan pengguna biasa. root akaun ini satu sahaja yang di sediakan dan ia di katakan sebagai "Super User". root ini selalu digunakan untuk mentadbir sistem anda.

Lain pula dengan pengguna biasa, ia adalah akaun yang selalu di gunakan untuk membuat kerja kerja harian dan tidak perlu menggunakan akses root.

4 - Pembahagian Struktur Cakera Keras

Struktur Cakera Keras yang saya pilih adalah yang saya buat sendiri. Anda boleh mengikut struktur yang di sediakan oleh skrip pemasangan sebagai permulaan. Swap partition adalah memori virtual komputer. Ia membolehkan sistem akses lebih dari memori kemampuan sebenar. Kebesarannya swap akan mengikut size fizikal memory atau dua kali ganda size fizikal.

5 - Komponen komponen OpenBSD

Di sini kita memilih semua komponen komponen OpenBSD yang ada untuk permasangan.

  • bsd - komponen kernel *perlu
  • bsd.mp - komponen kernel untuk muti prosessor
  • bsd.rd - komponen kernel ramdisk
  • base70.tgz - komponen utama
  • etc.tgz - file konfigurasi sistem utama
  • comp70.tgz - sistem kompiler, file library dan sebagainya
  • game70.tgz - game teks
  • xetc70.tgz - file konfigurasi x11
  • xserv.tgz - komponen X11
  • xbase70.tgz - komponen utama X11
  • xshare70.tgz - file ini perlu untuk xbase
  • xfont70.tgz - font X11

7 - Pertama kali Log Masuk ke OpenBSD

Sistem anda sudah boot ke dalam perisai OpenBSD dan jika anda terlihat ada url tertera http://firmware.openbsd.org, OpenBSD sedang memasang firmware drivers proprietary untuk perkakas anda. Dari tangkapan skrin di atas menunjukkan OpenBSD hendak memasang drivers untuk perkakas Intel. Kalau tidak ada access internet anda boleh menggunakan fw_update untuk memasang drivers tersebut apabila mendapat access internet.

Intel Firmware tidak dapat di edarkan bersama sama sistem operasi OpenBSD, ianya hanya boleh di muat turun apabila OpenBSD telah siap di pasang.

Saya log masuk dengan pengguna root sistem mengeluar kan mesej penyata

8 - Log masuk dari luar

OpenBSD dapat di akses dari luar. Maksud di sini kita log masuk ke sistem tidak berada di tempat sistem itu. kita memerlukan alamat IP atau alamat penuh URL ( seperti computer1.webluar.net) . Selain itu menggunakan aplikasi utility seperti:-

  • PuTTY - Aplikasi mudah terminal untuk
    perisai * Microsoft Windows menggunakan protokol SSH.
  • WinSCP - Aplikasi untuk memudahkan memindah file menggunakan SSH untuk perisian
    * Microsoft Windows menggunakan protokol SFTP, SCP.
  • ssh - Aplikasi tersebut tersedia di dalam perisai Linux, Apple macOS dan Microsoft Windows 10 ke atas
  • scp - Aplikasi memindah file tersedia di dalam perisai Linux dan Windows menggunakan protokol SSH

SSHD atau SSH Daemon ada lah service yang membolehkan kita log masuk dari luar.
