Konfigurasi OpenBSD

Selesai sahaja pemasang, kita akan membuat beberapa perubahan konfigurasi untuk OpenBSD yang lazim di buat.

doas pengganti sudo

Aplikasi doas membolehkan kita untuk melancarkan program program yang hanya dapat di lancar dari root dari pengguna biasa. doas mula di perkenalkan di dalam versi OpenBSD 5.8 sebagai pengganti aplikasi sudo. Fungsinya mirip seperti Run As Microsofot Windows.

cheesysalmon$ chown www&periodwww index.html
chown: index.html: Operation not permitted
cheesysalmon$ su ‐
Password:
cheesysalmon# echo 'permit persist keepenv badli' > /etc/doas.conf
cheesysalmon# cat /etc/doas.conf
permit persist keepenv badli
exit
cheesysalmon$ doas chown www.www index.html
doas (badli@cheesysalmon.my.domain) password:
cheesysalmon$ ls ‐al index.html
-rw-r--r--  1 www  www  0 Mar  5 12:03 index.html

File doas.conf telah di buat dengan teks untuk membolehkan saya melancarkan program chown sebagai pengguna root.

Menukar Nama Hos atau Nama Domain

Di dalam OpenBSD 7.0, kita perlu mengubah dua file /etc/hosts dan /etc/myname. Saya akan menukar cheesysalmon kepada cs kerana ada komputer telah mengambil nama cheesysalmon di dalam rangkaian kita. Fully Qualified Domain Name (FQDN) yang baru adalah cs.my.domain.Sudah selesai menukar nama nama tersebut kita perlu membuat sistem reboot untuk membuat perubahan tersebut.

Sekiranya hanya nama domain perlu di ubah seperti my.domain mestilah mengubah di dua dua file yang di maksudkan.

$ cat /etc/hosts
127.0.0.1             localhost
::1                        localhost
192.168.77.117  cheesysalmon.my.domain cheesysalmon

Nama hos ini penting di sebuah rangkaian komputer atau di Internet. Asas utama Internet adalah Sistem Nama Domain. Satu nama hos tidak boleh sama nama hos di rangkaian yang sama domain. Tetapi dua hos yang sama nama, berlainan rangkaian Internet domain ia ada lah berbeza. Contoh www.openbsd.org dan www.netbsd.org nama hos www sama tetapi nama domain berbeza.

Menukar Alamat IP, Subnet Mask Dan Gateway

Ada terdapat 2 jenis alamat IP yang di gunakan sekarang iaitu versi 4 dan 6. IPv4 mula di usahakan pada 1970'an. Request for Comment (RFC) pertama adalah RFC 791, Internet Protocol September 1981. Alamat IPv4 adalah 32 bit panjang dan boleh menampung sebanyak 4,294,967,296 secara teori tetapi bukan kesemua alamat IPv4 ini dapat dipakai.

Pengganti IPv4 adalah IPv6, populariti Internet makin meningkat pada 1990'an dan IPv4 dijangka bekurangan. IPv6 adalah 128 bit panjang dan boleh menampung sebanyak 340,282,366,920,938,463,463,374,607,431,768,211,465 alamat. Bermula dengan beberapa RFC yang telah di terbitkan pada tahun 1998 RFC 2460 (IPv6), 2461 (ipv6 Neighbour Discovery) dan 2463 (icmpv6).

IPv4 dan IPv6 adalah komponen komponen asas sistem rangkaian OpenBSD kerana BSD Unix adalah sistem operasi yang menjayakan IPv4 untuk ARPANET, dan OpenBSD telah di gunakan untuk menjayakan IPv6 untuk KAME projek. Transisi dari IPv4 dan IPv6 sedang giat diusahakan. Ini adalah report dari google mengenaie adoptasi IPv6 https://www.google.com/intl/en/ipv6/statistics.html
