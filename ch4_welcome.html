Korn Shell

Secara ringkas, Shell Unix adalah tempat kita berinteraksi dengan sistem Unix. Ia seumpamanya punat punat di kalkulator sedang menunggu petikan anda untuk mengira. Kiraan itu akan di paparkan di skrin kalkulator. Unix Shell yang di sediakan adalah Korn Shell atau nama singkatannya adalah ksh atau sh. Kalau di dalam perisaian Windows ia adalah 'Command Prompt' atau 'PowerShell'.

Pengedaran Korn Shell OpenBSD ini berasaskan kod kod Public Domain Korn Shell. Selain ksh terdapat juga C Shell atau singkatan csh. Kedua dua shell tersebut adalah komponen asas OpenBSD. Shell tersebut boleh di automasi untuk melancarkan dan mempermudahkan kerja kerja anda.

Jikalau anda suka pada shell yang lain seperti GNU Bash, Z shell atau mana mana Unix Shell yang lain anda perlu lah memasang Shell tersebut secara berasingan.

Direktori Home Anda - pwd dan echo

cheesysalmon$ pwd
/home/badli
cheesysalmon$ echo $HOME
/home/badli
cheesysalmon$

Kita mulakan dengan menaip pwd (singkatan Print Working Directory)dan tekan 'Enter' untuk memastikan kita berada di dalam home folder. Ia akan menunjukkan /home/badli di monitor andah.

Satu lagi cari untuk memastikan di mana home folder kita ialah dengan menaip echo $HOME. $HOME ada lah home environment variable. Setiap pengguna sistem ini mempunyai berlainan home direktori.

Di sini who memberitahu yang dua pengguna telah masuk ke sistem, saya dan jane.

Program hostname melihat kan nama sistem anda sahaja. tetapi uname -a akan memapar kan perisai yang di guna kan dan hostname

File Dan Direktori Yang Disembunyikan

cheesysalmon$ ls
cheesysalmon$ ls -a
. .. .Xdefaults .cshrc .cvsrc .login .mailrc .profile .ssh
cheesysalmon$

Taip ls (singkatan list) untuk melihat kandungan home direktori. Anda tidak nampak apa pun sama ada file atau direktori home kamu.

Sekarang taip ls -a kita akan mendapati ada file dan direktori yang di sembunyikan sebelumnya. -a di sini bermaksud untuk melihat semua file dan direktori walaupun ia di sembunyikan.

File dan direktori di sembunyikan tersebut adalah:-

  • .Xdefaults
  • .cshrc
  • .cvsrc
  • .login
  • .mailrc
  • .profile
  • .ssh

Kita boleh tahu yang mana satu file dan direktori dengan menaip ls -al. -l untuk menunjukkan butiran tersebut.

-rw-r--r-- 1 badli badli 212 Oct 1 04:00 .profile

.profile adalah file kerana di arah permission (-rw-r--r--) tidak ada terdapat huruf d yang menunjukkan ia adalah direktori.

drwx------ 2 badli badli 512 Feb 23 0344 .ssh

.ssh adalah direktori kerana di arah permission (drwx------) terdapat huruf d yang menunjukkan ia direktori. Kalau direktori ada huruf x ini membuatkan anda dapat memasuki direktori tersebut.

cheesysalmon$ file .profile
.profile: ASCII text
cheesysalmon$ file .ssh
.ssh: directory

Satu lagi kaedah untuk membeza di antara file dan direktori adalah dengan menggunakan file. Dari contoh program telah membezakan .profile adalah text file dan .ssh adalah folder.

cheesysalmon$ file /sys
/sys: broken symbolic link to 'usr/src/sys'
cheesysalmon$ file /dev/sd0a
/dev/sd1: block special (4/0)
cheesysalmon$ file /bin/ls
/bin/ls: ELF 64-bit LSB shared object, x86-64, version 1

Program file juga dapat memberi tahu beberapa jenis file.

  • /sys - adalah symbolic link, sama seperti shortcut di dalam Windows
  • /dev/sd0a - Ini ada block device untuk cakera keras anda
  • ls - adalah file binari yang kita lancarkan dari shell anda

tab completion

Dengan "tab completion" ia meringkas awda menaip program atau lokasi dengan arahan ke dalam ksh shell.

cheesysalmon$ cd /usr/
X11R6/    games/    lib/      libexec/  mdec/     sbin/     src/
bin/      include/  libdata/  local/    obj/      share/    xobj/

Cara menggunakan cd dengan menggunakan tab completion adalah seperti di atas. cd /u[tab] (tekan [tab]) akan menyempurnakan ke cd /usr dan cd /usr/[tab] akan melihatkan apa direktori yang ada di dalam /usr.

cheesysalmon$ z
zcat    zdiff   zegrep  zforce  zic     zmore   zzz
zcmp    zdump   zfgrep  zgrep   zless   znew

Kita juga boleh menggunakannya cara tab completion untuk command completion, contoh dengan menaip huruf z dan tekan tab. z[tab]. Seperti yang kita lihat di atas ia menunjukan semua program program yang bermula dengan huruf z.

Seperti sistem Unix yang lain nya anda boleh membaca dokumentasi program program atau arahan tersebut didalam OpenBSD sekiranya anda telah memasang man70.tgz semasa pemasangan. Kalau tidak di install anda boleh membuat permasang file tersebut.

Sistem pertolongan man

man bermaksud "perlihatkan manual", program ini program asas sistem pertolongan Unix. Jikalau anda pengguna sistem Unix, anda pasti pernah menggunakan man untuk mecari pilihan oleh program yang anda pakai. Dokumen dokumen yang di sediakah adalah bermutu tinggi. Format arahan man seperti,man program. Contoh man ls untuk melihat manual untuk ls.

Cara melihat dokumen tersebut adalah menggunakan butang arah dan page up dan page down key yang tersedia. Selesai memakai program man anda tekan q untuk quit. untuk sistem bantuan tekan h

GNU info

info adalah program sistem pertolongan GNU. Dokumentasi mereka menggunakan info format dan kebanyakan di edarkan dengan aplikasi GNU. Cara permakaian nya dengan menggunakan butang arah and q untuk quit

Program Dan Lokasinya

Lokasi program berada di dalam $PATH anda.

cheesysalmon$ echo $PATH
/home/badli/bin:/bin:/sbin:/usr/bin:/usr/sbin:/usr/X11R6/bin:/usr/local/bin:/usr/local/sbin:/usr/games
cheesysalmon$ ls -l /bin | wc -l
      42
cheesysalmon$ ls -l /sbin | wc -l
      75
cheesysalmon$ ls -l /usr/bin | wc -l
      345
cheesysalmon$ ls -l /usr/sbin | wc -l
206

$PATH ada lah tempat program program tersebut:-

  • /home/badli/bin
  • /bin
  • /sbin
  • /usr/bin
  • /usr/sbin
  • /usr/X11R6/bin
  • /usr/local/bin
  • /usr/local/sbin
  • /usr/games

¦ adalah symbol untuk mengambil output dari program ls -l itu and dijadikan input ke wc untuk mengira berapa program yang terdapat didalam sistem OpenBSD anda.
